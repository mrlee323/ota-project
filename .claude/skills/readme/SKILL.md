프로젝트를 분석해서 README.md를 생성하거나 정리합니다.

## 입력 처리

사용자 입력: `$ARGUMENTS`

- `--new`: 기존 README 무시하고 새로 작성
- `--update`: 기존 README에서 빠진 부분만 추가/수정
- `--ko`: 한국어로 작성 (기본값)
- `--en`: 영어로 작성
- 인자 없으면: 기존 README 있으면 업데이트, 없으면 새로 생성

## 1단계: 프로젝트 파악

다음 파일들을 읽어 프로젝트 정보를 수집하세요:

**필수 파일:**
- `package.json` / `pyproject.toml` / `Cargo.toml` — 프로젝트명, 버전, 의존성, 스크립트
- 현재 `README.md` — 기존 내용 파악 (있는 경우)
- 최상위 디렉토리 구조 (`ls -la`)

**추가 파악:**
- `src/` 또는 `app/` 최상위 구조 — 디렉토리 역할 파악
- `.env.example` — 필요한 환경 변수
- `docker-compose.yml` / `Dockerfile` — 배포/실행 방식
- `vite.config.*` / `next.config.*` / 기타 설정 파일 — 기술 스택 확인

기존 README가 있으면:
- 어떤 섹션이 있는지 파악
- 코드와 맞지 않는 내용 (오래된 스크립트명, 잘못된 경로 등) 체크
- 빠진 섹션이 무엇인지 파악

## 2단계: README 구성 결정

프로젝트 성격에 따라 섹션을 선택하세요.

**항상 포함:**
- 프로젝트 제목 + 한 줄 설명
- 기술 스택
- 시작하기 (설치 + 실행)

**해당되면 포함:**
- 스크린샷 / 데모 링크 (UI 프로젝트)
- 환경 변수 설정 (`.env.example` 있는 경우)
- 디렉토리 구조 (구조가 복잡한 경우)
- 주요 기능 (기능이 여러 개인 경우)
- API 문서 링크 또는 요약 (백엔드)
- 테스트 실행 방법 (테스트 설정 있는 경우)
- 배포 방법 (Dockerfile, CI/CD 있는 경우)
- 기여 가이드 (오픈소스 or 팀 프로젝트)

**포함하지 않아도 되는 것:**
- 코드에서 확인 안 되는 내용 (추측해서 작성 금지)
- 너무 뻔한 내용 ("이 프로젝트는 React로 만들어졌습니다")

## 3단계: README 작성

### 작성 원칙

- **실제 코드 기반**: `package.json`의 실제 스크립트명, 실제 포트 번호 사용
- **복사해서 바로 쓸 수 있는 명령어**: 설치·실행 명령어는 바로 붙여넣기 가능하게
- **간결하게**: 섹션당 핵심만, 장황한 설명 금지
- **추측 금지**: 확인되지 않은 내용은 `TODO:` 표시하거나 사용자에게 질문

## 기술 스택

| 분류 | 기술 |
|------|------|
| 프레임워크 | React 17 + Vite |
| 스타일링 | Emotion |
| 데이터 페칭 | React Query + Axios |
| 테스트 | Vitest + Testing Library |

## 시작하기

```bash
# 의존성 설치
yarn install

# 개발 서버 실행
yarn dev
# → http://localhost:5173
```

## 환경 변수 (해당하는 경우)

```bash
cp .env.example .env
```

| 변수명 | 설명 | 예시 |
|--------|------|------|
| `VITE_API_URL` | API 서버 주소 | `http://localhost:3000` |

## 디렉토리 구조 (복잡한 경우)

```
src/
├── pages/        # 라우트 단위 페이지
├── components/   # 재사용 컴포넌트
├── hooks/        # 커스텀 훅
├── utils/        # 유틸 함수
└── types/        # 타입 정의
```

## 주요 명령어

| 명령어 | 설명 |
|--------|------|
| `yarn dev` | 개발 서버 실행 |
| `yarn build` | 프로덕션 빌드 |
| `yarn test` | 테스트 실행 |
| `yarn preview` | 빌드 결과 미리보기 |

## 테스트 (테스트 있는 경우)

```bash
yarn test          # 전체 테스트
yarn test:watch    # watch 모드
```

## 4단계: 기존 README 업데이트 시 주의사항

기존 README가 있을 때:
- 사용자가 직접 작성한 **프로젝트 소개, 배경, 스크린샷**은 건드리지 않음
- 명령어, 경로, 버전 정보 등 **코드에서 확인 가능한 것**만 수정
- 삭제할 내용이 있으면 바로 지우지 말고 사용자에게 먼저 알림
